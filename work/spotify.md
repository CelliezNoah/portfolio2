Dans le cadre de mon exercice, j'ai créé un site de phishing en local imitant l'interface de Spotify. L'objectif était de comprendre comment fonctionnent ces attaques, en reproduisant le design et les fonctionnalités de la page de connexion de Spotify. Une fois que l'utilisateur entre ses informations sur le faux site, celles-ci sont envoyées à une base de données WAMP que j'ai mise en place. Cette base de données permet de récupérer les informations saisies, comme les identifiants de connexion.

Cet exercice m’a permis de mieux comprendre les mécanismes derrière les attaques de phishing et comment les données des utilisateurs peuvent être récupérées sans leur consentement. Toutefois, il est important de noter que ce genre d'exercice doit se faire dans un environnement sécurisé et contrôlé, uniquement à des fins pédagogiques, afin d'apprendre à se protéger contre ce type d'attaque.
